# Netology_DZ_Java_Queue

## Описание проекта

Учебный проект по Java для школы Нетология. Проект посвящен работе с очередями (Queue) и демонстрирует обработку клиентов маникюрного салона с вероятностным добавлением новых клиентов через рекомендации.

## Техническое задание

### Задача

Вы начинающий мастер по маникюру. Вы знаете, что делаете свою работу достаточно хорошо, так что каждый новый клиент с вероятностью 50% приведёт своего друга.

У вас есть список первых клиентов. Каждого клиента можно описать просто строкой:

```java
List<String> firstClients = List.of("Anya", "Sveta", "Olya", "Alexandra", "Ruslana", "Olesya", "Vika");
```

### Требования

1. Создать Queue-коллекцию из списка первых клиентов.
2. Каждого клиента принимать в порядке очереди.
3. В конце обработки каждого клиента с вероятностью 50% записывать его друга.
4. Имя другу генерировать из имени того, чьим другом он является через фразу "a friend of". Например:
   - Если порекомендовала Sveta, то друга будут звать "a friend of Sveta".
   - Если и её друг порекомендует, то такого друга будут звать "a friend of a friend of Sveta".
5. Новый записывающийся всегда встаёт в конец очереди.
6. В качестве обработки клиента выводить на экран фразу вида: `<имя> сделал новый маникюр`.

### Шаги реализации

1. Создать пустую очередь и заполнить её из списка `firstClients`.
2. Создать цикл разбора очереди клиентов (цикл `while` с проверкой очереди на непустоту).
3. На каждой итерации извлекать следующего клиента, обрабатывать его и затем пытаться добавить в очередь его друга.

## Описание решения

### Алгоритм работы

Программа использует структуру данных `Queue` (очередь) для хранения клиентов. Очередь реализована через `LinkedList`, что обеспечивает эффективное добавление элементов в конец и извлечение из начала.

**Основной алгоритм:**

1. **Инициализация**: Создается пустая очередь и заполняется из списка первых клиентов.
2. **Обработка**: В цикле `while` извлекается клиент из начала очереди.
3. **Выполнение услуги**: Выводится сообщение о выполненном маникюре.
4. **Вероятностное добавление друга**: С помощью `Math.random() < 0.5` проверяется вероятность 50%. Если условие выполняется, формируется имя друга по шаблону "a friend of <имя>" и добавляется в конец очереди.
5. **Повторение**: Цикл продолжается до тех пор, пока очередь не станет пустой.

### Особенности реализации

- Использование `Queue<String>` для хранения клиентов обеспечивает правильный порядок обработки (FIFO - First In, First Out).
- Метод `poll()` извлекает и удаляет элемент из начала очереди, возвращая `null` если очередь пуста.
- Метод `offer()` добавляет элемент в конец очереди.
- Проверка `!clientQueue.isEmpty()` гарантирует, что цикл завершится, когда все клиенты будут обработаны.
- Вероятность 50% реализована через `Math.random() < 0.5`, что дает равномерное распределение.

### Пример работы программы

При запуске программа может вывести, например:

```
Anya сделал новый маникюр.
Sveta сделал новый маникюр.
Olya сделал новый маникюр.
Alexandra сделал новый маникюр.
a friend of Anya сделал новый маникюр.
Ruslana сделал новый маникюр.
Olesya сделал новый маникюр.
a friend of Sveta сделал новый маникюр.
Vika сделал новый маникюр.
a friend of a friend of Anya сделал новый маникюр.
a friend of Olya сделал новый маникюр.

Все клиенты обработаны. Очередь пуста.
```

*Примечание: Результат может отличаться при каждом запуске из-за вероятностного характера добавления друзей.*

## Переменные и функции

### Класс `ManicureQueue`

#### Переменные

- **`firstClients`** (тип: `List<String>`)  
  Список первых клиентов маникюрного салона.  
  Содержит имена: "Anya", "Sveta", "Olya", "Alexandra", "Ruslana", "Olesya", "Vika".  
  Используется для инициализации очереди.

- **`clientQueue`** (тип: `Queue<String>`)  
  Очередь клиентов, ожидающих обработки.  
  Реализована через `LinkedList<String>`.  
  Хранит всех клиентов в порядке их обработки.

- **`currentClient`** (тип: `String`)  
  Текущий клиент, извлекаемый из очереди для обработки.  
  Используется для вывода сообщения и формирования имени друга (если он появится).

- **`friendName`** (тип: `String`)  
  Имя друга текущего клиента, формируемое по шаблону "a friend of <имя>".  
  Создается только при срабатывании вероятности 50% и добавляется в очередь.

#### Методы

- **`main(String[] args)`**  
  Главный метод программы, точка входа.  
  Выполняет инициализацию очереди, обработку клиентов и вероятностное добавление друзей.  
  **Параметры:** `args` - аргументы командной строки (не используются).  
  **Возвращаемое значение:** отсутствует (void).

#### Используемые методы стандартной библиотеки

- **`List.of(...)`** - создает неизменяемый список из переданных элементов.
- **`LinkedList<>()`** - конструктор для создания пустой очереди.
- **`queue.offer(element)`** - добавляет элемент в конец очереди.
- **`queue.poll()`** - извлекает и удаляет элемент из начала очереди, возвращает `null` если очередь пуста.
- **`queue.isEmpty()`** - проверяет, пуста ли очередь.
- **`Math.random()`** - генерирует случайное число от 0.0 (включительно) до 1.0 (исключительно).
